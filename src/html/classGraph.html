double Graph::twoOpt ( vInt &  path,
double  bestDistance 
)

Tour improvement algorithm to be ran after a solution has been found for the tsp problem Complexity: the complexity of this algorithm isn't clear. However, the lower bound is Ω(E*V³) where V is the number of vertexes and E the number of edges in the graph. The upper bound could be O(E*V³) as well, but a 2opt swap isn't guaranteed to not for a new intersection between edges

Parameters
pathtour considered in the algorithm that gave the solution to the tsp
bestDistancedistance that comes from a previous heuristic to find the solution for the tsp
Returns
the lowest distance of the path obtained with this algorithm
